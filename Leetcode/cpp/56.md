# 56. Merge Intervals — C++ Notes (push_back / emplace_back / move)

---

# 1. std::move

## 中文

`std::move` 的作用是：

> 將一個物件轉換成 **rvalue reference**，使其可以使用 **move semantics**。

重要觀念：

```
std::move 不會真的移動資料
它只是允許 move constructor 被呼叫
```

例如：

```cpp
vector<int> a = {1,2};
vector<int> b = std::move(a);
```

流程：

```
a ----move----> b
```

對 `vector` 來說，move 通常只是：

```
transfer pointer
```

時間複雜度：

```
move = O(1)
copy = O(n)
```

但副作用：

```
a 會變成 moved-from state
(仍然 valid，但內容未定)
```

---

## English

`std::move`:

> Casts an object to **rvalue reference** so that **move semantics** can be used.

Important concept:

```
std::move does NOT move anything by itself
It only enables move constructors
```

Example:

```cpp
vector<int> a = {1,2};
vector<int> b = std::move(a);
```

For vectors this usually means:

```
transfer pointer ownership
```

Complexity:

```
move = O(1)
copy = O(n)
```

Side effect:

```
a becomes a moved-from object
(still valid but unspecified state)
```

# 2. push_back

## 中文

`push_back` 用於：

```
插入一個已存在的 object
```

例子：

```cpp
vector<int> a = {1,2};
results.push_back(a);
```

流程：

```
copy constructor
```

如果使用：

```cpp
results.push_back(std::move(a));
```

流程：

```
move constructor
```

---

## English

`push_back` is used to:

```
insert an existing object
```

Example:

```cpp
vector<int> a = {1,2};
results.push_back(a);
```

Process:

```
copy constructor
```

With move:

```cpp
results.push_back(std::move(a));
```

Process:

```
move constructor
```

# 3. emplace_back

## 中文

`emplace_back` 的概念：

> **直接在 container memory 中建構物件**

例如：

```cpp
vector<pair<int,int>> v;

v.emplace_back(1,2);
```

流程：

```
vector memory
    ↓
construct pair(1,2)
```

沒有 temporary object。

---

## English

`emplace_back`:

> Constructs the object **directly inside the container memory**.

Example:

```cpp
vector<pair<int,int>> v;
v.emplace_back(1,2);
```

Process:

```
vector memory
    ↓
construct pair(1,2)
```

No temporary object is created.

---

# 4. push_back vs emplace_back

## 中文

| 情況          | 建議           |
| ----------- | ------------ |
| 已有 object   | push_back    |
| 需要建構 object | emplace_back |

例子：

```
push_back(x)     插入 existing object
emplace_back(a,b) 直接建構 object
```

---

## English

| Scenario             | Recommended  |
| -------------------- | ------------ |
| Existing object      | push_back    |
| Construct new object | emplace_back |

Example:

```
push_back(x)      insert existing object
emplace_back(a,b) construct object in place
```

# 5. Temporary Object

## 中文

temporary object 會出現在：

```
push_back(Type(...))
push_back({...})
push_back("hello")  // implicit conversion
```

例如：

```cpp
v.push_back({1,2});
```

流程：

```
temporary pair
     ↓
move/copy
     ↓
vector
```

---

## English

Temporary objects occur when:

```
push_back(Type(...))
push_back({...})
push_back("hello")
```

Example:

```cpp
v.push_back({1,2});
```

Process:

```
temporary object
     ↓
move/copy
     ↓
vector
```

# 6. Merge Intervals Example

Original code:

```cpp
results.push_back(move(intervals[i]));
```

Options:

| Code                             | Behavior |
| -------------------------------- | -------- |
| push_back(intervals[i])          | copy     |
| push_back(move(intervals[i]))    | move     |
| emplace_back(intervals[i])       | copy     |
| emplace_back(move(intervals[i])) | move     |

Therefore:

```
push_back(move(x))
=
emplace_back(move(x))
```

---

# 7. Recommended Code for This Problem

Because each interval is small:

```
vector<int> size = 2
```

Copy cost is negligible.

Most engineers prefer:

```cpp
results.push_back(intervals[i]);
```

Reasons:

```
1. simpler code
2. no moved-from state
3. negligible copy cost
```

# 8. Key Takeaways

## 中文

```
std::move 不會真的移動資料
push_back 用於插入 existing object
emplace_back 用於直接建構 object
move 比 copy 快 (O(1) vs O(n))
```

---

## English

```
std::move does not move data itself
push_back inserts existing objects
emplace_back constructs objects in place
move is usually faster than copy (O(1) vs O(n))
```
